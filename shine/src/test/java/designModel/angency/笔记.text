代理模式
1、解决的问题：将次要业务、主要业务解耦合的过程
2、次要业务和主要业务的区别
   次要业务：起到辅助作用，辅助主要业务顺利实现，在项目中往往大量存在

3.次要业务对开发效率影响：
     jdbc开发步骤
     1）加载驱动 【次要业务】
     2）建立连接通道 【次要业务】
     3）建立数据库操作对象 【次要业务】
     4）推送sql命令到数据库执行并返回处理结果 【主要业务】
     5）销毁con、statement、rd   【次要业务】


4、代理模式的本质：行为的监听   代理对象 $proxy InvocationHandler

5、代理模式组成：
   1）接口：申明需要被监听的行为
   2）代理实现类（InvocationHandler）：次要业务 次要业务和主要业务绑定执行
   3）代理对象（监听对象）


6、业务：饭前便后要洗手
